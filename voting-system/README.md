# Voting System

## Requirements

You must design a real-time voting system with the following requirements:

1. Never lose data.
1. Be secure and prevent bots and malicious actors.
1. Handle 300 million users.
1. Handle a peak of 250,000 requests per second (RPS).
1. Ensure users can vote only once.
1. Should be real-time.

You can't use:

1. Serverless.
1. MongoDB.
1. On-premise.
1. Google Cloud.
1. Azure.
1. OpenShift.
1. Mainframes.
1. Monolithic solutions.
